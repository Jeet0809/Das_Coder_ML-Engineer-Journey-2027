## 🏗️ Project Structure Standard

Each project will follow this structure:

project-name/
├── README.md (comprehensive)
├── requirements.txt
├── data/ (with README)
├── notebooks/ (exploration)
├── src/ (production code)
├── tests/ (unit tests)
├── deployment/ (Docker, K8s)
├── docs/ (architecture, API)
└── app/ (web interface)
